# 01 — 바로읽기 / 읽고 싶은 책 저장 액션

## 개요

책 상세 화면의 두 CTA 버튼에 대한 비즈니스 로직.
서재에 책을 추가하고 상태를 설정하는 핵심 액션이다.

**의존**: `00-swiftdata`, `03-book-detail`

---

## 사용자 시나리오

### 시나리오 A — 바로 읽기
1. 책 상세에서 [바로 읽기] 탭
2. 서재에 해당 책이 없으면 `reading` 상태로 추가
3. 이미 `wantToRead` 상태이면 `reading`으로 업데이트
4. 탭 1(독서 탭)로 이동
5. 해당 책의 타이머 자동 시작

### 시나리오 B — 읽고 싶은 책 저장
1. 책 상세에서 [읽고 싶은 책에 저장] 탭
2. 서재에 해당 책이 없으면 `wantToRead` 상태로 추가
3. 성공 피드백 (버튼 상태 변경)

### 시나리오 C — 저장 취소 (토글)
1. 이미 `wantToRead` 상태인 책에서 버튼 탭
2. 서재에서 제거 (삭제 확인 불필요)
3. 버튼 상태 복원

---

## 기능 명세

### 바로 읽기 규칙
- `completed` 상태인 책은 [바로 읽기] 버튼 비활성 (다시 읽기 기능은 Phase 3)
- `reading` 상태인 책은 [읽고 있는 중] 표시, 탭 시 독서 탭으로 이동만
- 서재 추가 실패 시 에러 알럿 표시

### 중복 추가 방지
- `fetchBy(isbn:)` 로 존재 여부 선확인
- 이미 있으면 추가하지 않고 상태만 업데이트

## 작업 목록

### 1. 액션 로직 (ViewModel or UseCase)
- [ ] `readNow(book: Book)` 구현
  - [ ] ISBN으로 서재 존재 여부 확인
  - [ ] 없으면 `LibraryBook` 생성 + `reading` 상태로 저장
  - [ ] 있고 `wantToRead`이면 `reading`으로 업데이트
  - [ ] 탭 1 전환 트리거 (`selectedTab` 바인딩 또는 노티)
  - [ ] 타이머 자동 시작 트리거 (탭 1로 이동 후 해당 책 타이머 오픈)
- [ ] `saveToWishlist(book: Book)` 구현
  - [ ] ISBN으로 서재 존재 여부 확인
  - [ ] 없으면 `wantToRead` 상태로 저장
- [ ] `removeFromWishlist(isbn: String)` 구현
  - [ ] 서재에서 해당 책 삭제

### 2. 탭 전환 연동
- [ ] `MainView`의 `selectedTab`을 환경 또는 바인딩으로 하위 뷰까지 전달
- [ ] `readNow` 완료 후 탭 1로 전환

### 3. 타이머 자동 시작 연동
- [ ] 독서 탭 진입 시 특정 책의 타이머를 자동 오픈하는 트리거 설계
  - 방법 A: `@AppStorage` / `UserDefaults`에 "열어야 할 bookId" 임시 저장
  - 방법 B: `TimerMainView`에 `pendingBookId` 상태 주입
- [ ] 선택한 방법으로 구현

### 4. 에러 처리
- [ ] 저장 실패 시 `Alert` 표시

## 완료 기준

- [ ] [바로 읽기] 탭 시 서재에 `reading` 상태로 추가되고 탭 1로 이동
- [ ] [읽고 싶은 책에 저장] 탭 시 `wantToRead` 상태로 추가됨
- [ ] 동일 ISBN 책은 중복 추가되지 않음
- [ ] 이미 저장된 책의 버튼 상태가 상세 화면 진입 시 올바르게 표시됨
- [ ] 저장 취소(토글) 시 서재에서 삭제됨
