# 05 — 타이머 ↔ 세션 저장 연동

## 개요

기존 `TimerView`에 세션 저장 기능을 연결한다.
타이머 시작 시 세션을 시작하고, 나가기(종료) 시 `ReadingSession`으로 저장한다.
이 기능이 완료되어야 독서 시간 통계(Phase 2)의 기반 데이터가 쌓인다.

**의존**: `00-swiftdata`, `04-reading-tab`

---

## 사용자 시나리오

1. 독서 탭에서 책 카드의 [이어 읽기] 탭
2. 타이머 fullScreenCover 오픈, 선택한 책 정보 표시
3. 타이머 자동 시작 (또는 [시작] 버튼 탭)
4. 독서 중 일시정지 / 재개 가능
5. [나가기] 탭 → 타이머 중지
6. 현재까지의 독서 시간이 `ReadingSession`으로 저장됨
7. 독서 탭으로 복귀, 카드의 누적 시간이 업데이트됨

---

## 기능 명세

### 세션 저장 조건
- 최소 1분 이상 독서한 경우에만 세션 저장 (노이즈 방지)
- 1분 미만이면 저장하지 않고 그냥 종료
- 저장 실패 시 에러 로그 (UI 에러는 표시하지 않음)

### 나가기 플로우
1. [나가기] 탭
2. 타이머 정지
3. 독서 시간 ≥ 1분이면 세션 저장
4. fullScreenCover dismiss

### 타이머 화면 표시 정보
- 선택된 책 제목 (상단)
- 타이머 (시:분:초)
- 시작 / 일시정지 / 재개 버튼
- [나가기] 버튼 (상단 또는 하단)

---

## 데이터

- 입력: `LibraryBook` (선택된 책)
- 저장: `ReadingSession(id, libraryBookId, startTime, endTime, duration)`
- `LibraryRepositoryProtocol.addSession(_:to:)`

---

## 작업 목록

### 1. TimerView에 책 정보 연결
- [ ] `TimerView`가 `LibraryBook`을 받도록 인터페이스 수정
- [ ] 화면 상단에 책 제목 표시

### 2. 세션 저장 로직
- [ ] `TimerViewModel`(또는 기존 로직)에 `startTime: Date` 기록 추가
- [ ] [나가기] 액션에 세션 저장 로직 추가
  - [ ] `endTime = Date()`
  - [ ] `duration = elapsedTime` (기존 타이머 경과 시간)
  - [ ] 1분 이상이면 `addSession` 호출
- [ ] 저장 완료 후 `dismiss`

### 3. 타이머 자동 시작
- [ ] `TimerView` 진입 시 자동 시작 옵션 파라미터 추가 (`autoStart: Bool`)
- [ ] `readNow` 액션에서 `autoStart: true`로 전달

### 4. 나가기 확인 (선택)
- [ ] 타이머 실행 중 나가기 시 확인 Alert ("독서를 종료하고 저장할까요?")
- [ ] 확인 → 저장 후 종료 / 취소 → 타이머 계속

---

## 완료 기준

- [ ] 타이머 종료(나가기) 시 `ReadingSession`이 DB에 저장됨
- [ ] 1분 미만 독서는 세션이 저장되지 않음
- [ ] 세션 저장 후 독서 탭 카드의 누적 시간이 업데이트됨
- [ ] 일시정지 중 나가도 일시정지까지의 시간으로 세션 저장됨
- [ ] 동일 책으로 여러 세션 저장이 정상 동작함
